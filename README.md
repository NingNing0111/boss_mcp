# 陪你成长的招聘助理

## MVP 设计

本项目参考 GenericAgent 的设计，目标是实现一个具备自演化能力的招聘 Agent。

Agent 的核心能力是：在持续对话中理解用户的职业背景、能力特征、求职偏好与阶段性目标，并基于这些信息帮助用户发现、筛选、评估并沟通合适的岗位。

## 已实现能力

目前已完成两个核心平台能力接入：

### 1. 企查查

用于提供企业信息查询能力。

代码位置：

```text
src/qcc/handler/*.rs
```

已实现能力包括：

* 登录状态检查
* 扫码登录
* 企业名称模糊搜索
* 企业详情查询

### 2. Boss 直聘

用于提供岗位搜索、岗位详情获取、HR 初次沟通与聊天消息处理能力。

代码位置：

```text
src/boss/handler/*.rs
```

已实现能力包括：

* 登录状态检查
* 扫码登录
* 岗位搜索
* 岗位详情获取
* 立即开聊
* 未读聊天列表获取
* 聊天详情获取
* 招呼语发送

# 招聘助理核心能力

## 企查查能力说明

### `login_check.rs`

用于检查企查查登录状态。

在调用企查查任何功能前，都必须先执行登录检查，避免因未登录导致后续流程失败。

### `login.rs`

用于实现企查查扫码登录。

调用后返回登录二维码，用户扫码后即可完成登录。

### `search_company.rs`

根据企业名称进行模糊搜索，返回企业列表。

通常情况下，列表中排在最前面的企业与搜索词相关度最高。

### `company_detail.rs`

根据企业信息获取企业详情。

可用于辅助判断目标公司是否适合用户，例如公司规模、经营状态、风险信息、行业属性等。

### 推荐调用流程

```text
login_check -> login -> search_company -> company_detail
```

### 注意事项

企查查相关流程均在有头浏览器中执行，因此实现时需要关注当前浏览器所在页面。

例如：

* 当前是否停留在登录页
* 是否已经完成扫码登录
* 是否需要跳转到搜索页
* 详情页是否由搜索结果页进入
* 页面状态是否会影响后续 handler 执行

## Boss 直聘能力说明

### `login_check.rs`

用于检查 Boss 直聘登录状态。

在调用 Boss 直聘任何功能前，都必须先执行登录检查，避免因未登录影响岗位搜索、开聊或消息读取流程。

### `login.rs`

用于实现 Boss 直聘扫码登录。

调用后返回登录二维码，用户扫码后即可完成登录。

### `search_position.rs`

用于搜索岗位。

搜索参数通过 `job_query.rs` 构建，返回匹配到的岗位信息列表。

### `position_detail.rs`

用于获取具体岗位详情。

该模块同时提供“立即开聊”能力。调用 `start_chat` 后，页面会进入消息页，并自动打开对应岗位的聊天框。

### `unread_chat.rs`

用于处理消息页的聊天数据。

包含两个核心方法：

* `get_unread_chat`：获取消息页的聊天列表信息
* `get_unread_chat_message`：获取指定聊天会话中的对话详情数据

调用 `get_unread_chat_message` 时，会触发并打开对应聊天框，之后即可发送消息。

### `send_greeting_message.rs`

用于向已打开的聊天框发送消息。

发送消息的前提是聊天框已经被触发并打开。

聊天框可通过以下两种方式触发：

1. 调用 `position_detail.rs` 中的 `start_chat`
2. 调用 `unread_chat.rs` 中的 `get_unread_chat_message`

聊天框打开后，即可调用 `send_greeting_message` 发送自定义招呼语或沟通内容。

# 产品目标

招聘助理的长期目标是成为一个“陪用户成长”的求职 Agent。

它不只是一次性搜索岗位，而是能够在每一次对话中持续总结和提炼用户信息，包括：

* 职业经历
* 技能栈
* 项目经验
* 行业偏好
* 城市偏好
* 薪资期望
* 岗位方向
* 沟通风格
* 职业目标
* 当前求职阶段

基于这些信息，Agent 应逐步构建用户画像，并持续优化岗位匹配策略。

最终目标是让 AI 能够：

1. 理解用户适合什么岗位；
2. 判断岗位是否值得投递或沟通；
3. 分析目标公司的可靠性与匹配度；
4. 自动生成合适的开场沟通内容；
5. 根据 HR 反馈持续调整求职策略；
6. 在长期交互中不断完善用户职业画像。
